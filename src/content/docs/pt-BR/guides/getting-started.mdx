O Biome é melhor instalado como uma dependência de desenvolvimento dos seus projetos, mas também está disponível como um [executável standalone] que não requer Node.js.

[executável standalone]: /guides/manual-installation

<PackageManagerCommand
  npm="i -D -E @biomejs/biome"
  pnpm="add -D -E @biomejs/biome"
  yarn="add -D -E @biomejs/biome"
  bun="add -D -E @biomejs/biome"
  deno="add -D npm:@biomejs/biome"
/>

:::note[Fixação de versão]
`-E` garante que o gerenciador de pacotes fixe a versão do Biome. Consulte a
[página de versionamento](/internals/versioning)
para mais informações sobre [por que fixar a versão é importante].
:::

[por que fixar a versão é importante]: /internals/versioning#why-pinning-the-version-is-important

## Configuração

Embora o Biome possa executar sem configuração alguma, você provavelmente vai querer ajustar algumas configurações para atender às necessidades do seu projeto, e nesse caso você pode executar o seguinte comando para gerar um arquivo de configuração `biome.json`.

<PackageManagerBiomeCommand command="init"/>

## Uso

Vamos ter uma visão geral rápida de como usar o Biome no seu projeto.

### Interface de linha de comando

O Biome fornece uma [interface de linha de comando] para formatar, fazer lint e verificar seu código.

[interface de linha de comando]: /reference/cli

### Integrações com editores

O Biome está disponível como uma extensão oficial nos seus editores favoritos.

- [VS Code](/editors/first-party-extensions#vs-code)
- [IntelliJ](/editors/first-party-extensions#intellij)
- [Zed](/editors/first-party-extensions#zed)

Também existem [extensões da comunidade](/editors/third-party-extensions)
para outros editores, como **Vim**, **Neovim** e **Sublime Text**, para citar alguns.

### Integração Contínua

Execute `biome ci` como parte do seu pipeline de CI para garantir qualidade e consistência do código em toda a sua equipe. Funciona exatamente como o comando `biome check`, mas é otimizado para ambientes de CI.

- [GitHub Actions](/recipes/continuous-integration#github-actions)
- [GitLab CI](/recipes/continuous-integration#gitlab-ci)

Consulte as receitas de [Integração Contínua](/recipes/continuous-integration) para mais exemplos.
