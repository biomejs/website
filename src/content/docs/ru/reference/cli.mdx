## biome

Официальный CLI Biome. Используйте его для проверки состояния вашего проекта или запуска проверки отдельных файлов.

**Использование**: **`biome`** _`COMMAND ...`_

**Доступные опции:**
- **`-h`**, **`--help`** &mdash;
  Выводит справочную информацию
- **`-V`**, **`--version`** &mdash;
  Выводит информацию о версии



**Доступные команды:**
- **`version`** &mdash;
  Показывает информацию о версии Biome и завершает работу.
- **`rage`** &mdash;
  Выводит информацию для отладки.
- **`start`** &mdash;
  Запускает процесс демона-сервера Biome.
- **`stop`** &mdash;
  Останавливает процесс демона-сервера Biome.
- **`check`** &mdash;
  Запускает форматирование, линтинг и сортировку импортов для указанных файлов.
- **`lint`** &mdash;
  Выполняет различные проверки для набора файлов.
- **`format`** &mdash;
  Запускает форматирование для набора файлов.
- **`ci`** &mdash;
  Команда для использования в CI окружениях. Запускает форматирование, линтинг и сортировку импортов для указанных файлов.
- **`init`** &mdash;
  Инициализирует новый проект biome. Создает файл конфигурации с некоторыми настройками по умолчанию.
- **`lsp-proxy`** &mdash;
  Действует как сервер для Language Server Protocol через stdin/stdout.
- **`migrate`** &mdash;
  Обновляет конфигурацию при наличии критических изменений.
- **`search`** &mdash;
  ЭКСПЕРИМЕНТАЛЬНО: Ищет паттерны Grit в проекте.
- **`explain`** &mdash;
  Показывает документацию по различным аспектам CLI.
- **`clean`** &mdash;
  Очищает логи, создаваемые демоном.


## biome version

Показывает информацию о версии Biome и завершает работу.

**Использование**: **`biome`** **`version`**

**Глобальные опции, применяемые ко всем командам**
- **`    --colors`**=_`<off|force>`_ &mdash;
  Устанавливает режим форматирования разметки: "off" выводит все как обычный текст, "force" принудительно форматирует разметку с использованием ANSI, даже если вывод консоли определен как несовместимый
- **`    --use-server`** &mdash;
  Подключается к запущенному экземпляру демона-сервера Biome.
- **`    --verbose`** &mdash;
  Выводит дополнительную диагностику, некоторые диагностические сообщения показывают больше информации. Также выводит, какие файлы были обработаны и какие были изменены.
- **`    --config-path`**=_`PATH`_ &mdash;
  Устанавливает путь к файлу конфигурации или путь к директории для поиска `biome.json` или `biome.jsonc`. При использовании отключает стандартное разрешение файла конфигурации.

  Использует переменную окружения **`BIOME_CONFIG_PATH`**
- **`    --max-diagnostics`**=_`<none|<NUMBER>>`_ &mdash;
  Ограничивает количество отображаемых диагностических сообщений. Когда указано `none`, лимит снимается.

  [по умолчанию: 20]
- **`    --skip-parse-errors`** &mdash;
  Пропускает файлы, содержащие синтаксические ошибки, вместо вывода диагностического сообщения об ошибке.
- **`    --no-errors-on-unmatched`** &mdash;
  Подавляет ошибки, которые были бы выведены, если бы ни один файл не был обработан во время выполнения команды.
- **`    --error-on-warnings`** &mdash;
  Указывает Biome завершиться с кодом ошибки, если некоторые диагностические сообщения содержат предупреждения.
- **`    --reporter`**=_`<json|json-pretty|github|junit|summary|gitlab|checkstyle|rdjson>`_ &mdash;
  Позволяет изменить способ вывода диагностики и сводки.
- **`    --log-file`**=_`ARG`_ &mdash;
  Необязательный путь для перенаправления логов.

  Если не указан, логи выводятся в stdout.
- **`    --log-level`**=_`<none|debug|info|warn|error>`_ &mdash;
  Уровень логирования. В порядке от самого подробного до наименее подробного: debug, info, warn, error.

  Значение `none` отключает любое логирование.

  [по умолчанию: none]
- **`    --log-kind`**=_`<pretty|compact|json>`_ &mdash;
  Формат вывода логов.

  [по умолчанию: pretty]
- **`    --diagnostic-level`**=_`<info|warn|error>`_ &mdash;
  Уровень диагностики для отображения. В порядке от наименее важного к наиболее важному: info, warn, error. Передача `--diagnostic-level=error` приведет к тому, что Biome будет выводить только диагностику, содержащую только ошибки.

  [по умолчанию: info]



**Доступные опции:**
- **`-h`**, **`--help`** &mdash;
  Выводит справочную информацию

## biome check

Запускает форматирование, линтинг и сортировку импортов для указанных файлов.

**Использование**: **`biome`** **`check`** \[**`--write`**\] \[**`--unsafe`**\] \[**`--assist-enabled`**=_`<true|false>`_\] \[**`--enforce-assist`**=_`<true|false>`_\] \[**`--format-with-errors`**=_`<true|false>`_\] \[**`--staged`**\] \[**`--changed`**\] \[**`--since`**=_`REF`_\] \[_`PATH`_\]...

**Опции, изменяющие поведение JSON парсера**
- **`    --json-parse-allow-comments`**=_`<true|false>`_ &mdash;
  Разрешает разбор комментариев в `.json` файлах
- **`    --json-parse-allow-trailing-commas`**=_`<true|false>`_ &mdash;
  Разрешает разбор завершающих запятых в `.json` файлах



**Конфигурация, содержащаяся в файле `biome.json`**
- **`    --vcs-enabled`**=_`<true|false>`_ &mdash;
  Должен ли Biome интегрироваться с клиентом VCS
- **`    --vcs-client-kind`**=_`<git>`_ &mdash;
  Тип клиента.
- **`    --vcs-use-ignore-file`**=_`<true|false>`_ &mdash;
  Должен ли Biome использовать файл игнорирования VCS. Когда [true], Biome будет игнорировать файлы, указанные в файле игнорирования.
- **`    --vcs-root`**=_`PATH`_ &mdash;
  Папка, где Biome должен проверять наличие файлов VCS. По умолчанию Biome будет использовать ту же папку, где был найден `biome.json`.

  Если Biome не может найти конфигурацию, он попытается использовать текущую рабочую директорию. Если текущая рабочая директория не может быть найдена, Biome не будет использовать интеграцию с VCS, и будет выведено диагностическое сообщение
- **`    --vcs-default-branch`**=_`BRANCH`_ &mdash;
  Основная ветка проекта
- **`    --files-max-size`**=_`NUMBER`_ &mdash;
  Максимально допустимый размер файлов исходного кода в байтах. Файлы, превышающие этот лимит, будут игнорироваться по причинам производительности. По умолчанию 1 МиБ
- **`    --files-ignore-unknown`**=_`<true|false>`_ &mdash;
  Указывает Biome не выводить диагностику при обработке файлов, которые он не знает
- **`    --format-with-errors`**=_`<true|false>`_ &mdash;
  Должно ли форматирование продолжаться, если данный файл имеет синтаксические ошибки
- **`    --indent-style`**=_`<tab|space>`_ &mdash;
  Стиль отступов.
- **`    --indent-width`**=_`NUMBER`_ &mdash;
  Размер отступа, по умолчанию 2
- **`    --line-ending`**=_`<lf|crlf|cr|auto>`_ &mdash;
  Тип окончания строки.
- **`    --line-width`**=_`NUMBER`_ &mdash;
  Максимальная ширина строки. По умолчанию 80.
- **`    --attribute-position`**=_`<multiline|auto>`_ &mdash;
  Стиль позиции атрибутов в HTML-подобных языках. По умолчанию auto.
- **`    --bracket-same-line`**=_`<true|false>`_ &mdash;
  Размещать `>` многострочного HTML или JSX элемента в конце последней строки вместо того, чтобы он был на следующей строке (не применяется к самозакрывающимся элементам).
- **`    --bracket-spacing`**=_`<true|false>`_ &mdash;
  Вставлять ли пробелы вокруг скобок в объектных литералах. По умолчанию true.
- **`    --expand`**=_`<auto|always|never>`_ &mdash;
  Раскрывать ли массивы и объекты на несколько строк. Когда установлено `auto`, объектные литералы форматируются на несколько строк, если первое свойство имеет перевод строки, а массивы форматируются на одну строку, если умещаются. Когда установлено `always`, эти литералы форматируются на несколько строк независимо от длины списка. Когда установлено `never`, эти литералы форматируются на одну строку, если умещаются. При форматировании `package.json`, Biome будет использовать `always`, если не настроено иначе. По умолчанию "auto".
- **`    --use-editorconfig`**=_`<true|false>`_ &mdash;
  Использовать любые файлы `.editorconfig` для настройки форматирования. Конфигурация в `biome.json` переопределит конфигурацию `.editorconfig`.

  По умолчанию: `true`.
- **`    --jsx-everywhere`**=_`<true|false>`_ &mdash;
  Когда включено, файлы типа `.js`/`.mjs`/`.cjs` могут содержать синтаксис JSX.

  По умолчанию `true`.
- **`    --javascript-formatter-enabled`**=_`<true|false>`_ &mdash;
  Управляет форматированием для JavaScript (и его надъязыков) файлов.
- **`    --jsx-quote-style`**=_`<double|single>`_ &mdash;
  Тип кавычек, используемых в JSX. По умолчанию double.
- **`    --quote-properties`**=_`<preserve|as-needed>`_ &mdash;
  Когда свойства в объектах берутся в кавычки. По умолчанию asNeeded.
- **`    --trailing-commas`**=_`<all|es5|none>`_ &mdash;
  Выводить завершающие запятые везде, где это возможно, в многострочных структурах с разделителями-запятыми. По умолчанию "all".
- **`    --semicolons`**=_`<always|as-needed>`_ &mdash;
  Выводит ли форматер точки с запятой для всех операторов или только в for операторах, где это необходимо из-за ASI.
- **`    --arrow-parentheses`**=_`<always|as-needed>`_ &mdash;
  Добавлять ли необязательные скобки к стрелочным функциям. По умолчанию "always".
- **`    --bracket-same-line`**=_`<true|false>`_ &mdash;
  Размещать ли закрывающую скобку многострочных HTML/JSX тегов в конце последней строки, а не на следующей строке. По умолчанию false.
- **`    --javascript-formatter-indent-style`**=_`<tab|space>`_ &mdash;
  Стиль отступа, применяемый к JavaScript (и его надъязыкам) файлам.
- **`    --javascript-formatter-indent-width`**=_`NUMBER`_ &mdash;
  Размер отступа, применяемый к JavaScript (и его надъязыкам) файлам. По умолчанию 2.
- **`    --javascript-formatter-line-ending`**=_`<lf|crlf|cr|auto>`_ &mdash;
  Тип окончания строки, применяемый к JavaScript (и его надъязыкам) файлам. `auto` использует CRLF на Windows и LF на других платформах.
- **`    --javascript-formatter-line-width`**=_`NUMBER`_ &mdash;
  Максимальная ширина строки, применяемая к JavaScript (и его надъязыкам) файлам. По умолчанию 80.
- **`    --javascript-formatter-quote-style`**=_`<double|single>`_ &mdash;
  Тип кавычек, используемых в коде JavaScript. По умолчанию double.
- **`    --javascript-formatter-attribute-position`**=_`<multiline|auto>`_ &mdash;
  Стиль позиции атрибутов в JSX элементах. По умолчанию auto.
- **`    --javascript-formatter-bracket-spacing`**=_`<true|false>`_ &mdash;
  Вставлять ли пробелы вокруг скобок в объектных литералах. По умолчанию true.
- **`    --javascript-formatter-expand`**=_`<auto|always|never>`_ &mdash;
  Раскрывать ли массивы и объекты на несколько строк. Когда установлено `auto`, объектные литералы форматируются на несколько строк, если первое свойство имеет перевод строки, а массивы форматируются на одну строку, если умещаются. Когда установлено `always`, эти литералы форматируются на несколько строк независимо от длины списка. Когда установлено `never`, эти литералы форматируются на одну строку, если умещаются. При форматировании `package.json`, Biome будет использовать `always`, если не настроено иначе. По умолчанию "auto".
- **`    --javascript-formatter-operator-linebreak`**=_`<before|after>`_ &mdash;
  При разбиении бинарных выражений на несколько строк, разбивать их до или после бинарного оператора. По умолчанию "after".
- **`    --javascript-linter-enabled`**=_`<true|false>`_ &mdash;
  Управляет линтером для JavaScript (и его надъязыков) файлов.
- **`    --javascript-assist-enabled`**=_`<true|false>`_ &mdash;
  Управляет помощником для JavaScript (и его надъязыков) файлов.
- **`    --json-parse-allow-comments`**=_`<true|false>`_ &mdash;
  Разрешает разбор комментариев в `.json` файлах
- **`    --json-parse-allow-trailing-commas`**=_`<true|false>`_ &mdash;
  Разрешает разбор завершающих запятых в `.json` файлах
- **`    --json-formatter-enabled`**=_`<true|false>`_ &mdash;
  Управляет форматированием для JSON (и его надъязыков) файлов.
- **`    --json-formatter-indent-style`**=_`<tab|space>`_ &mdash;
  Стиль отступа, применяемый к JSON (и его надъязыкам) файлам.
- **`    --json-formatter-indent-width`**=_`NUMBER`_ &mdash;

  Исправляет нарушения правил линтера с помощью подавления комментариями вместо использования действия правила кода (исправления)
- **`    --reason`**=_`STRING`_ &mdash;
  Объяснение для подавления диагностики с `--suppress`
- **`    --only`**=_`<GROUP|RULE|DOMAIN>`_ &mdash;
  Запускает только заданное правило, группу правил или домен. Если уровень серьезности правила `off`, то уровень серьезности правила устанавливается в `error`, если это рекомендуемое правило, или `warn` в противном случае.

  Пример:


  ```shell
  biome lint --only=correctness/noUnusedVariables --only=suspicious --only=test
  ```

- **`    --skip`**=_`<GROUP|RULE|DOMAIN>`_ &mdash;
  Пропускает заданное правило, группу правил или домен, устанавливая уровень серьезности правил в `off`. Эта опция имеет приоритет над `--only`.

  Пример:


  ```shell
  biome lint --skip=correctness/noUnusedVariables --skip=suspicious --skip=project
  ```

- **`    --stdin-file-path`**=_`PATH`_ &mdash;
  Используйте эту опцию, когда хотите отформатировать код, переданный из `stdin`, и вывести результат в `stdout`.

  Файл не обязан существовать на диске, важно расширение файла. На основе расширения Biome знает, как проверить код линтером.

  Пример: ```shell echo 'let a;' | biome lint --stdin-file-path=file.js --write ```
- **`    --staged`** &mdash;
  Когда установлено в true, будут проверены только файлы, которые были добавлены в индекс (подготовлены для коммита).
- **`    --changed`** &mdash;
  Когда установлено в true, будут проверены только файлы, которые были изменены по сравнению с вашей конфигурацией `defaultBranch`.
- **`    --since`**=_`REF`_ &mdash;
  Используйте это, чтобы указать базовую ветку для сравнения, когда вы используете флаг --changed и `defaultBranch` не установлен в вашем biome.json
- **`-h`**, **`--help`** &mdash;
  Выводит справочную информацию


## biome format

Запускает форматирование для набора файлов.

**Использование**: **`biome`** **`format`** \[**`--write`**\] \[**`--staged`**\] \[**`--changed`**\] \[**`--since`**=_`REF`_\] \[_`PATH`_\]...

**Общие опции, применяемые ко всем файлам**
- **`    --format-with-errors`**=_`<true|false>`_ &mdash;
  Должно ли форматирование продолжаться, если данный файл имеет синтаксические ошибки
- **`    --indent-style`**=_`<tab|space>`_ &mdash;
  Стиль отступов.
- **`    --indent-width`**=_`NUMBER`_ &mdash;
  Размер отступа, по умолчанию 2
- **`    --line-ending`**=_`<lf|crlf|cr|auto>`_ &mdash;
  Тип окончания строки.
- **`    --line-width`**=_`NUMBER`_ &mdash;
  Максимальная ширина строки. По умолчанию 80.
- **`    --attribute-position`**=_`<multiline|auto>`_ &mdash;
  Стиль позиции атрибутов в HTML-подобных языках. По умолчанию auto.
- **`    --bracket-same-line`**=_`<true|false>`_ &mdash;
  Размещать `>` многострочного HTML или JSX элемента в конце последней строки вместо того, чтобы он был на следующей строке (не применяется к самозакрывающимся элементам).
- **`    --bracket-spacing`**=_`<true|false>`_ &mdash;
  Вставлять ли пробелы вокруг скобок в объектных литералах. По умолчанию true.
- **`    --expand`**=_`<auto|always|never>`_ &mdash;
  Раскрывать ли массивы и объекты на несколько строк. Когда установлено `auto`, объектные литералы форматируются на несколько строк, если первое свойство имеет перевод строки, а массивы форматируются на одну строку, если умещаются. Когда установлено `always`, эти литералы форматируются на несколько строк независимо от длины списка. Когда установлено `never`, эти литералы форматируются на одну строку, если умещаются. При форматировании `package.json`, Biome будет использовать `always`, если не настроено иначе. По умолчанию "auto".
- **`    --use-editorconfig`**=_`<true|false>`_ &mdash;
  Использовать любые файлы `.editorconfig` для настройки форматирования. Конфигурация в `biome.json` переопределит конфигурацию `.editorconfig`.

  По умолчанию: `true`.



**Опции форматирования, специфичные для JavaScript файлов**
- **`    --javascript-formatter-enabled`**=_`<true|false>`_ &mdash;
  Управляет форматированием для JavaScript (и его надъязыков) файлов.
- **`    --jsx-quote-style`**=_`<double|single>`_ &mdash;
  Тип кавычек, используемых в JSX. По умолчанию double.
- **`    --quote-properties`**=_`<preserve|as-needed>`_ &mdash;

- **`    --trailing-commas`**=_`<all|es5|none>`_ &mdash;
  Выводить завершающие запятые везде, где это возможно, в многострочных структурах с разделителями-запятыми. По умолчанию "all".
- **`    --semicolons`**=_`<always|as-needed>`_ &mdash;
  Выводит ли форматер точки с запятой для всех операторов или только в for операторах, где это необходимо из-за ASI.
- **`    --arrow-parentheses`**=_`<always|as-needed>`_ &mdash;

- **`    --bracket-same-line`**=_`<true|false>`_ &mdash;
  Размещать ли закрывающую скобку многострочных HTML/JSX тегов в конце последней строки, а не на следующей строке. По умолчанию false.
- **`    --javascript-formatter-indent-style`**=_`<tab|space>`_ &mdash;
  Стиль отступа, применяемый к JavaScript (и его надъязыкам) файлам.
- **`    --javascript-formatter-indent-width`**=_`NUMBER`_ &mdash;
  Размер отступа, применяемый к JavaScript (и его надъязыкам) файлам. По умолчанию 2.
- **`    --javascript-formatter-line-ending`**=_`<lf|crlf|cr|auto>`_ &mdash;
  Тип окончания строки, применяемый к JavaScript (и его надъязыкам) файлам. `auto` использует CRLF на Windows и LF на других платформах.
- **`    --javascript-formatter-line-width`**=_`NUMBER`_ &mdash;
  Максимальная ширина строки, применяемая к JavaScript (и его надъязыкам) файлам. По умолчанию 80.
- **`    --javascript-formatter-quote-style`**=_`<double|single>`_ &mdash;
  Тип кавычек, используемых в коде JavaScript. По умолчанию double.
- **`    --javascript-formatter-attribute-position`**=_`<multiline|auto>`_ &mdash;
  Стиль позиции атрибутов в JSX элементах. По умолчанию auto.
- **`    --javascript-formatter-bracket-spacing`**=_`<true|false>`_ &mdash;
  Вставлять ли пробелы вокруг скобок в объектных литералах. По умолчанию true.

Когда установлено `auto`, объектные литералы форматируются на несколько строк, если первое свойство имеет перевод строки, а массивы форматируются на одну строку, если умещаются. ## biome search

ЭКСПЕРИМЕНТАЛЬНО: Ищет паттерны Grit в проекте.

Примечание: GritQL экранирует фрагменты кода с помощью обратных кавычек, но большинство оболочек интерпретируют обратные кавычки как вызовы команд. Чтобы избежать этого, лучше всего заключить ваши запросы Grit в одинарные кавычки.

### Пример

  ```shell
  biome search '`console.log($message)`' # найти все вызовы `console.log`
  ```


**Использование**: **`biome`** **`search`** \[**`-l`**=_`ARG`_\] _`PATTERN`_ \[_`PATH`_\]...

**Глобальные опции, применяемые ко всем командам**

  [по умолчанию: info]



**Конфигурация файловой системы**
- **`    --files-max-size`**=_`NUMBER`_ &mdash;
  Максимально допустимый размер файлов исходного кода в байтах. Файлы, превышающие этот лимит, будут игнорироваться по причинам производительности. По умолчанию 1 МиБ
- **`    --files-ignore-unknown`**=_`<true|false>`_ &mdash;
  Указывает Biome не выводить диагностику при обработке файлов, которые он не знает



**Набор свойств для интеграции Biome с программным обеспечением VCS.**
- **`    --vcs-enabled`**=_`<true|false>`_ &mdash;
  Должен ли Biome интегрироваться с клиентом VCS

  Папка, где Biome должен проверять наличие файлов VCS. По умолчанию Biome будет использовать ту же папку, где был найден `biome.json`.

  Если Biome не может найти конфигурацию, он попытается использовать текущую рабочую директорию. Если текущая рабочая директория не может быть найдена, Biome не будет использовать интеграцию с VCS, и будет выведено диагностическое сообщение
- **`    --vcs-default-branch`**=_`BRANCH`_ &mdash;
  Основная ветка проекта



**Доступные позиционные элементы:**
- _`PATTERN`_ &mdash;
  Паттерн GritQL для поиска.

  Обратите внимание, что команда search (в настоящее время) не поддерживает переписывание.
- _`PATH`_ &mdash;
  Один файл, один путь или список путей.



**Доступные опции:**
- **`    --stdin-file-path`**=_`PATH`_ &mdash;
  Используйте эту опцию, когда хотите выполнить поиск в коде, переданном из `stdin`, и вывести результат в `stdout`.

  Файл не обязан существовать на диске, важно расширение файла. На основе расширения Biome знает, как разобрать код.

  Пример: ```shell echo 'let a;' | biome search '`let $var`' --stdin-file-path=file.js ```
- **`-l`**, **`--language`**=_`ARG`_ &mdash;
  Язык, к которому применяется паттерн.

  Запросы Grit специфичны для грамматики языка, на который они нацелены, поэтому в настоящее время мы не поддерживаем написание запросов, которые применяются к нескольким языкам одновременно.

  Когда не указано, языком по умолчанию является JavaScript.

  Выводит справочную информацию
