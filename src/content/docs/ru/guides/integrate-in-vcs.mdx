---
title: Интеграция Biome с вашей VCS
description: Узнайте, как вы можете интегрировать Biome с VCS
---

Интеграция VCS (система контроля версий) предназначена для использования **дополнительных** функций, которые может предоставить только VCS.
На данный момент Biome поддерживает только Git.
Интеграция является **опциональной**.
Вам необходимо включить `vcs.enabled` и установить `vcs.clientKind` в конфигурационном файле Biome:

```json title="biome.json"
{
  "vcs": {
    "enabled": true,
    "clientKind": "git"
  }
}
```

Эта конфигурация **сама по себе ничего не делает**. Вам нужно включить функции, которые вы хотите использовать.

### Использование файла игнорирования

Включите `vcs.useIgnoreFile`, чтобы позволить Biome игнорировать все файлы и директории, перечисленные в файле игнорирования VCS проекта, а также файл `.ignore`.

```json title="biome.json" ins={5}
{
  "vcs": {
    "enabled": true,
    "clientKind": "git",
    "useIgnoreFile": true
  }
}
```

### Обработка только измененных файлов

Это функция, доступная только через CLI, и позволяет обрабатывать **только** те файлы, которые **изменились** от одной ревизии к другой.

Сначала вам нужно обновить ваш конфигурационный файл и сообщить Biome, какая ветка является ветвью по умолчанию, через поле `vcs.defaultBranch`:

```json title="biome.json" ins={6}
{
  "vcs": {
    "enabled": true,
    "clientKind": "git",
    "useIgnoreFile": true,
    "defaultBranch": "main"
  }
}
```

Затем добавьте опцию `--changed` к вашей команде, чтобы обработать только те файлы, которые ваша VCS признала как "измененные". Biome, с помощью VCS, определит измененные файлы от ветки `main` до вашей текущей ревизии:

```shell
biome check --changed
```

:::caution
Biome не проверяет, что именно изменилось, это означает, что даже добавление пробелов или переносов строк в файл пометит этот файл как "измененный"
:::

Альтернативно, вы можете использовать опцию `--since`, чтобы указать произвольную ветку. Эта опция **имеет приоритет** над опцией `vcs.defaultBranch`. Например, вы можете захотеть проверить свои изменения относительно ветки `next`:

```shell
biome check --changed --since=next
```

### Обработка только подготовленных файлов

Перед фиксацией ваших изменений вы можете захотеть проверить форматирование и линтинг файлов, которые были добавлены в _индекс_, также известных как _подготовленные файлы_.
Добавьте опцию `--staged` к вашей команде, чтобы обработать только эти файлы:

```shell
biome check --staged
```

Опция `--staged` недоступна в команде `ci`, потому что вы не должны фиксировать изменения в среде CI.
