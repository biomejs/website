## biome

CLI oficial de Biome. Utilízala para comprobar el estado de tu proyecto o ejecútala para comprobar archivos individuales.

**Uso**: **`biome`** _`COMANDO ...`_

**Opciones disponibles:**

- **`-h`**, **`--help`** &mdash;
  Imprime información de ayuda
- **`-V`**, **`--version`** &mdash;
  Imprime información sobre la versión

**Comandos disponibles:**

- **`version`** &mdash;
  Muestra la información de la versión de Biome.
- **`rage`** &mdash;
  Imprime información para depuración.
- **`start`** &mdash;
  Inicia el proceso del servidor daemon de Biome.
- **`stop`** &mdash;
  Detiene el proceso del servidor daemon de Biome.
- **`check`** &mdash;
  Ejecuta el formateador, linter y clasificación de importación a los archivos solicitados.
- **`lint`** &mdash;
  Ejecutar varias comprobaciones en un conjunto de archivos.
- **`format`** &mdash;
  Ejecutar el formateador en un conjunto de archivos.
- **`ci`** &mdash;
  Comando para utilizar en entornos CI. Ejecuta formateador, linter y clasificación de importación a los archivos solicitados.
- **`init`** &mdash;
  Crea un nuevo proyecto biome. Crea un archivo de configuración con algunos valores predeterminados.
- **`lsp-proxy`** &mdash;
  Actúa como servidor para el Protocolo de Servidor de Lenguas a través de stdin/stdout.
- **`migrate`** &mdash;
  Actualiza la configuración cuando se producen cambios de última hora.
- **`search`** &mdash;
  EXPERIMENTAL: Búsqueda de patrones de Grit en un proyecto.
- **`explain`** &mdash;
  Muestra documentación sobre diversos aspectos del CL.
- **`clean`** &mdash;
  Limpia los logs emitidos por el daemon.

## versión de biome

Muestra la información de la versión de Biome.

**Uso**: **`biome`** **`version`**

**Opciones globales aplicadas a todos los comandos**

- **`    --colors`**=_`<off|force>`_ &mdash;
  Establece el modo de formateo de las marcas: "off" imprime todo como texto plano, “force” fuerza el formateo de markup usando ANSI incluso si se determina que la salida de la consola es incompatible
- **`    --use-server`** &mdash;
  Conéctate a una instancia en ejecución del servidor daemon de Biome.
- **`    --verbose`** &mdash;
  Imprime diagnósticos adicionales, y algunos diagnósticos muestran más información. Además, imprime qué archivos se procesaron y cuáles se modificaron.
- **`    --config-path`**=_`PATH`_ &mdash;
  Establece la ruta del archivo de configuración, o la ruta del directorio para encontrar `biome.json` o `biome.jsonc`. Si se utiliza, desactiva la resolución del archivo de configuración por defecto.
- **`    --max-diagnostics`**=_`<none|<NUMBER>>`_ &mdash;
  Limita la cantidad de diagnósticos mostrados. Si se indica "none", se elimina el límite.
  [default: 20]
- **`    --skip-errors`** &mdash;
  Omitir los archivos que contienen errores de sintaxis en lugar de emitir un diagnóstico de error.
- **`    --no-errors-on-unmatched`** &mdash;
  Silenciar los errores que se emitirían en caso de que no se procesara ningún archivo durante la ejecución del comando.
- **`    --error-on-warnings`** &mdash;
  Indicar a Biome que salga con un código de error si algunos diagnósticos emiten advertencias.
- **`    --reporter`**=_`<json|json-pretty|github|junit|summary|gitlab>`_ &mdash;
  Permite cambiar la forma en que se comunican los diagnósticos y el resumen.
- **`    --log-level`**=_`<none|debug|info|warn|error>`_ &mdash;
  El nivel de registro. En orden, del más verboso al menos verboso: debug, info, warn, error.

  El valor `none` no mostrará ningún registro.

  [default: none]

- **`    --log-kind`**=_`<pretty|compact|json>`_ &mdash;
  Cómo debería ser el registro.
  [default: pretty]
- **`    --diagnostic-level`**=_`<info|warn|error>`_ &mdash;
  El nivel de diagnóstico a mostrar. En orden, del más bajo al más importante: info, warn, error. Si se pasa `--diagnostic-level=error`, Biome sólo imprimirá los diagnósticos que contengan errores.
  [default: info]

**Opciones disponibles:**

- **`-h`**, **`--help`** &mdash;
  Imprime información de ayuda

## biome start

Inicia el proceso del servidor daemon de Biome.

**Uso**: **`biome`** **`start`** \[**`--config-path`**=_`PATH`_\]

**Opciones disponibles:**

- **`    --log-prefix-name`**=_`STRING`_ &mdash;
  Permite cambiar el prefijo aplicado al nombre de archivo de los registros.
  Utiliza la variable de entorno **`BIOME_LOG_PREFIX_NAME`**
  [default: server.log]
- **`    --log-path`**=_`PATH`_ &mdash;
  Permite cambiar la carpeta donde se almacenan los registros.
  Utiliza la variable de entorno **`BIOME_LOG_PATH`**
- **`    --config-path`**=_`PATH`_ &mdash;
  Permite establecer una ruta de archivo personalizada para el archivo de configuración, o una ruta de directorio personalizada para encontrar `biome.json` o `biome.jsonc`.
  Utiliza la variable de entorno **`BIOME_CONFIG_PATH`**
- **`-h`**, **`--help`** &mdash;
  Imprime información de ayuda

**La configuración contenida en el archivo `biome.json`**

- **`    --vcs-enabled`**=_`<true|false>`_ &mdash;
  Si Biome debe integrarse con el cliente VCS
- **`    --vcs-client-kind`**=_`<git>`_ &mdash;
  El tipo de cliente.
- **`    --vcs-use-ignore-file`**=_`<true|false>`_ &mdash;
  Si Biome debe utilizar el archivo de ignorar VCS. Si es [true], Biome ignorará los archivos especificados en el archivo de ignorar.
- **`    --vcs-root`**=_`PATH`_ &mdash;
  La carpeta en la que Biome debe buscar los archivos VCS. Por defecto, Biome utilizará la misma carpeta en la que se encontró `biome.json`.

  Si Biome no puede encontrar la configuración, intentará utilizar el directorio de trabajo actual. Si no se puede encontrar el directorio de trabajo actual, Biome no utilizará la integración VCS y se emitirá un diagnóstico

- **`    --vcs-default-branch`**=_`BRANCH`_ &mdash;
  La rama principal del proyecto
- **`    --files-max-size`**=_`NUMBER`_ &mdash;
  El tamaño máximo permitido para los archivos de código fuente en bytes. Los archivos que superen este límite serán ignorados por razones de rendimiento. Por defecto 1 MiB
- **`    --files-ignore-unknown`**=_`<true|false>`_ &mdash;
  Indica a Biome que no emita diagnósticos al manipular archivos que no conoce
- **`    --use-editorconfig`**=_`<true|false>`_ &mdash;
  Utiliza cualquier archivo `.editorconfig` para configurar el formateador. La configuración en `biome.json` anulará la configuración de `.editorconfig`. Por defecto: false.
- **`    --indent-style`**=_`<tab|space>`_ &mdash;
  El estilo de sangría.
- **`    --indent-size`**=_`NUMBER`_ &mdash;
  El tamaño de la sangría, 2 por defecto (obsoleto, usa `indent-width`)
- **`    --indent-width`**=_`NUMBER`_ &mdash;
  El tamaño de la sangría, 2 por defecto
- **`    --line-ending`**=_`<lf|crlf|cr>`_ &mdash;
  El tipo de final de línea.
- **`    --line-width`**=_`NUMBER`_ &mdash;
  Cuál es el ancho máximo de una línea. Por defecto es 80.
- **`    --attribute-position`**=_`<multiline|auto>`_ &mdash;
  El estilo de posición del atributo en lenguajes HTMLish. Por defecto auto.
- **`    --bracket-spacing`**=_`<true|false>`_ &mdash;
  Si se insertan espacios entre paréntesis en los literales de los objetos. Por defecto es true.
- **`    --jsx-quote-style`**=_`<double|single>`_ &mdash;
  El tipo de comillas utilizadas en JSX. Por defecto es doble.
- **`    --quote-properties`**=_`<preserve|as-needed>`_ &mdash;
  Cuando se citan las propiedades de los objetos. Por defecto es asNeeded.
- **`    --trailing-comma`**=_`<all|es5|none>`_ &mdash;
  Imprime las comas finales siempre que sea posible en estructuras sintácticas de varias líneas separadas por comas. Por defecto, "all".
- **`    --trailing-commas`**=_`<all|es5|none>`_ &mdash;
  Imprime las comas finales siempre que sea posible en estructuras sintácticas de varias líneas separadas por comas. Por defecto, "all".
- **`    --semicolons`**=_`<always|as-needed>`_ &mdash;

- **`    --arrow-parentheses`**=_`<always|as-needed>`_ &mdash;
  Si se añaden paréntesis no necesarios a las funciones de flecha. Por defecto es "always".
- **`    --bracket-same-line`**=_`<true|false>`_ &mdash;

- **`    --unsafe`** &mdash;
  Permite realizar correcciones no seguras, debe usarse con `--write` o `--fix`
- **`    --fix`** &mdash;
  Alias para `--write`, escribe correcciones seguras, formateo y ordenación de importaciones
- **`    --apply`** &mdash;
  Alias para `--write`, escribe correcciones seguras, formateo y ordenación de importaciones (obsoleto, use `--write`)
- **`    --apply-unsafe`** &mdash;

- **`    --organize-imports-enabled`**=_`<true|false>`_ &mdash;

  Permite activar o desactivar las asistencias.
- **`    --stdin-file-path`**=_`PATH`_ &mdash;
  Utiliza esta opción cuando quieras formatear el código enviado desde `stdin`, e imprimir la salida en `stdout`.

  No es necesario que el archivo exista en el disco, lo que importa es la extensión del archivo. Basándose en la extensión, Biome sabe cómo comprobar el código.

  Ejemplo: `echo 'let a;' | biome check --stdin-file-path=file.js`

- **`    --staged`** &mdash;
  Cuando se establece en true, sólo los archivos que han sido puestos en escena (los que están preparados para ser confirmados) serán lintados. Esta opción debe utilizarse cuando se trabaja localmente.
- **`    --changed`** &mdash;

  Indica a Biome que no emita diagnósticos al manipular archivos que no conoce

**Opciones específicas del linter JavaScript**

- **`    --error-on-warnings`** &mdash;
