Desde la versión **v1.8.0**, la CLI de Biome acepta un argumento `--reporter` que permite cambiar cómo se imprimen los diagnósticos y el resumen en el terminal.

## Resumen

```shell
biome check --reporter=summary
```

## JSON

:::caution
Este generador de informe es experimental y está sujeto a cambios en las versiones de los parches.
:::

Emite el resumen y los diagnósticos en formato JSON.

```shell
biome ci --reporter=json
```

## JSON pretty

:::caution
Este generador de informe es experimental y está sujeto a cambios en las versiones de los parches.
:::

Igual que `--reporter=json`, emite el resumen y los diagnósticos en formato JSON, y la salida se formatea utilizando las opciones de formato JSON actuales (archivo de configuración o valores predeterminados).

```shell
biome ci --reporter=json-pretty
```

## GitHub

Utiliza este generador de informe en un flujo de trabajo de GitHub. Cuando se configura correctamente en un flujo de trabajo PR, GitHub imprimirá un mensaje por cada info/advertencia/error emitido.

```shell
biome ci --reporter=github
```

```
::error title=lint/suspicious/noDoubleEquals,file=main.ts,line=4,endLine=4,col=3,endColumn=5::Use === instead of ==
::error title=lint/suspicious/noDebugger,file=main.ts,line=6,endLine=6,col=1,endColumn=9::This is an unexpected use of the debugger statement.
::error title=lint/nursery/noEvolvingAny,file=main.ts,line=8,endLine=8,col=5,endColumn=6::This variable's type is not allowed to evolve implicitly, leading to potential any types.
```
