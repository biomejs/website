З версії **v1.8.0**, CLI Biome приймає аргумент `--reporter`, який дозволяє змінювати спосіб виведення діагностики та підсумків у термінал.

biome check --reporter=summary

## JSON

:::caution
Цей репортер є експериментальним і може змінюватися в патч-релізах.
:::

Виводить підсумок і діагностику у форматі JSON.

```shell
biome ci --reporter=json
```

## JSON pretty

:::caution
Цей репортер є експериментальним і може змінюватися в патч-релізах.
:::

Те саме, що і `--reporter=json`, він виводить підсумок і діагностику у форматі JSON, а вихідні дані форматуються за допомогою поточних параметрів форматування JSON (файл конфігурації або значення за замовчуванням).

```shell
biome ci --reporter=json-pretty
```

## GitHub

Використовуйте цей репортер у GitHub workflow. При правильній конфігурації у PR workflow, GitHub виведе повідомлення для кожного info/warning/error, що виникає.

```shell
biome ci --reporter=github
```

```
::error title=lint/suspicious/noDoubleEquals,file=main.ts,line=4,endLine=4,col=3,endColumn=5::Use === instead of ==
::error title=lint/suspicious/noDebugger,file=main.ts,line=6,endLine=6,col=1,endColumn=9::This is an unexpected use of the debugger statement.

```
