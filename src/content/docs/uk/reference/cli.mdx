## biome

Офіційний CLI Biome. Використовуйте його для перевірки стану вашого проєкту або для перевірки окремих файлів.

**Використання**: **`biome`** _`COMMAND ...`_

**Доступні опції:**

-   **`-h`**, **`--help`** &mdash;
    Вивести інформацію про допомогу
-   **`-V`**, **`--version`** &mdash;
    Вивести інформацію про версію

**Доступні команди:**

-   **`version`** &mdash;
    Показати інформацію про версію Biome та вийти.
-   **`rage`** &mdash;
    Вивести інформацію для налагодження.
-   **`start`** &mdash;
    Запустити процес серверу Biome daemon.
-   **`stop`** &mdash;
    Зупинити процес серверу Biome daemon.
-   **`check`** &mdash;
    Запустити форматування, лінтинг та сортування імпортів для запитаних файлів.
-   **`lint`** &mdash;
    Запустити різні перевірки для набору файлів.
-   **`format`** &mdash;
    Запустити форматувальник для набору файлів.
-   **`ci`** &mdash;
    Команда для використання в CI середовищах. Запускає форматування, лінтинг та сортування імпортів для запитаних файлів.
-   **`init`** &mdash;
    Ініціалізувати новий проєкт Biome. Створює конфігураційний файл з деякими налаштуваннями за замовчуванням.
-   **`lsp-proxy`** &mdash;
    Працює як сервер для Language Server Protocol через stdin/stdout.
-   **`migrate`** &mdash;
    Оновлює конфігурацію при наявності змін, що порушують сумісність.
-   **`search`** &mdash;
    ЕКСПЕРИМЕНТАЛЬНО: Шукає шаблони Grit по всьому проєкту.
-   **`explain`** &mdash;
    Показує документацію різних аспектів CLI.
-   **`clean`** &mdash;
    Очищає логи, створені daemon-ом.

**Глобальні опції, що застосовуються до всіх команд**

-   **`    --colors`**=_`<off|force>`_ &mdash;
    Встановити режим форматування для розмітки: "off" друкує все як звичайний текст, "force" примушує форматування розмітки за допомогою ANSI, навіть якщо консольний вихід визначено як несумісний
-   **`    --use-server`** &mdash;
    Підключитися до запущеного екземпляра серверу Biome daemon.
-   **`    --verbose`** &mdash;
    Друкувати додаткові діагностики, а деякі діагностики показують більше інформації. Також друкувати, які файли були оброблені та які з них були змінені.
-   **`    --config-path`**=_`PATH`_ &mdash;
    Встановити шлях до конфігураційного файлу або шлях до директорії для пошуку `biome.json` або `biome.jsonc`. Якщо використовується, вимикає стандартне вирішення конфігураційного файлу.
-   **`    --max-diagnostics`**=_`<none|<NUMBER>>`_ &mdash;
    Обмежити кількість відображуваних діагностик. Коли вказано `none`, ліміт знімається.
    [за замовчуванням: 20]
-   **`    --skip-errors`** &mdash;
    Пропускати файли з синтаксичними помилками замість видачі діагностики помилки.
-   **`    --no-errors-on-unmatched`** &mdash;
    Приглушити помилки, які будуть видані у випадку, якщо жоден файл не був оброблений під час виконання команди.
-   **`    --error-on-warnings`** &mdash;
    Вказати Biome виходити з кодом помилки, якщо деякі діагностики видають попередження.
-   **`    --reporter`**=_`<json|json-pretty|github|junit|summary|gitlab>`_ &mdash;
    Дозволяє змінити спосіб звітування діагностик та підсумків.
-   **`    --log-level`**=_`<none|debug|info|warn|error>`_ &mdash;
    Рівень логування. У порядку від найбільш детального до найменш детального: debug, info, warn, error.

    Значення `none` не показуватиме жодного логування.

    [за замовчуванням: none]

-   **`    --log-kind`**=_`<pretty|compact|json>`_ &mdash;
    Як повинен виглядати лог.
    [за замовчуванням: pretty]
-   **`    --diagnostic-level`**=_`<info|warn|error>`_ &mdash;
    Рівень діагностик для відображення. У порядку від найнижчого до найважливішого: info, warn, error. Передача `--diagnostic-level=error` призведе до того, що Biome друкуватиме лише діагностики, що містять лише помилки.
    [за замовчуванням: info]

**Доступні позиційні елементи:**

-   _`PATH`_ &mdash;
    Окремий файл, окремий шлях або список шляхів

**Доступні опції:**

-   **`    --write`** &mdash;
    Записує безпечні виправлення, форматування та сортування імпортів
-   **`    --unsafe`** &mdash;
    Дозволити небезпечні виправлення, слід використовувати з `--write` або `--fix`
-   **`    --fix`** &mdash;
    Псевдонім для `--write`, записує безпечні виправлення, форматування та сортування імпортів
-   **`    --apply`** &mdash;
    Псевдонім для `--write`, записує безпечні виправлення, форматування та сортування імпортів (застаріле, використовуйте `--write`)
-   **`    --apply-unsafe`** &mdash;
    Псевдонім для `--write --unsafe`, записує безпечні та небезпечні виправлення, форматування та сортування імпортів (застаріле, використовуйте `--write --unsafe`)
-   **`    --formatter-enabled`**=_`<true|false>`_ &mdash;
    Дозволити або заборонити перевірку форматувальника.
-   **`    --linter-enabled`**=_`<true|false>`_ &mdash;
    Дозволити або заборонити перевірку лінтера.
-   **`    --organize-imports-enabled`**=_`<true|false>`_ &mdash;
    Дозволити або заборонити сортування імпортів.
-   **`    --assists-enabled`**=_`<true|false>`_ &mdash;
    Дозволити або заборонити асисти.
-   **`    --stdin-file-path`**=_`PATH`_ &mdash;
    Використовуйте цю опцію, коли хочете форматувати код, переданий через `stdin`, і вивести результат на `stdout`.

    Файл не обов'язково має існувати на диску, важливе розширення файлу. Виходячи з розширення, Biome знає, як перевірити код.

    Приклад: `echo 'let a;' | biome check --stdin-file-path=file.js`

-   **`    --staged`** &mdash;
    Коли встановлено true, будуть перевірені лише файли, які були підготовлені до коміту. Цю опцію слід використовувати під час локальної роботи.
-   **`    --changed`** &mdash;
    Коли встановлено true, будуть перевірені лише файли, які були змінені порівняно з вашою конфігурацією `defaultBranch`. Цю опцію слід використовувати в CI середовищах.
-   **`    --since`**=_`REF`_ &mdash;
    Використовуйте це для вказівки базової гілки для порівняння, коли ви використовуєте прапорець --changed і `defaultBranch` не встановлено у вашому `biome.json`
-   **`-h`**, **`--help`** &mdash;
    Друкує інформацію про допомогу

## biome lint

Запустити різні перевірки для набору файлів.

**Використання**: **`biome`** **`lint`** \[**`--write`**\] \[**`--unsafe`**\] \[**`--only`**=_`<GROUP|RULE>`_\]... \[**`--skip`**=_`<GROUP|RULE>`_\]... \[**`--staged`**\] \[**`--changed`**\] \[**`--since`**=_`REF`_\] \[_`PATH`_\]...

**Набір властивостей для інтеграції Biome з VCS програмним забезпеченням.**

-   **`    --vcs-enabled`**=_`<true|false>`_ &mdash;
    Чи повинен Biome інтегруватися з VCS клієнтом
-   **`    --vcs-client-kind`**=_`<git>`_ &mdash;
    Тип клієнта.
-   **`    --vcs-use-ignore-file`**=_`<true|false>`_ &mdash;
    Чи повинен Biome використовувати файл ігнорування VCS. Коли [true], Biome буде ігнорувати файли, зазначені у файлі ігнорування.
-   **`    --vcs-root`**=_`PATH`_ &mdash;
    Папка, де Biome повинен перевіряти файли VCS. За замовчуванням Biome використовуватиме ту саму папку, де було знайдено `biome.json`.

    Якщо Biome не може знайти конфігурацію, він спробує використовувати поточну робочу директорію. Якщо поточна робоча директорія не може бути знайдена, Biome не використовуватиме інтеграцію VCS, і буде видана діагностика

-   **`    --vcs-default-branch`**=_`BRANCH`_ &mdash;
    Основна гілка проєкту

**Конфігурація файлової системи**

-   **`    --files-max-size`**=_`NUMBER`_ &mdash;
    Максимально допустимий розмір файлів вихідного коду в байтах. Файли, що перевищують цей ліміт, будуть ігноруватися з міркувань продуктивності. За замовчуванням 1 МіБ
-   **`    --files-ignore-unknown`**=_`<true|false>`_ &mdash;
    Вказує Biome не видавати діагностики при обробці файлів, які він не знає

**Опції лінтера, специфічні для JavaScript лінтера**

-   **`    --javascript-linter-enabled`**=_`<true|false>`_ &mdash;
    Контролювати лінтер для JavaScript (та його надмов) файлів.

**Опції лінтера, специфічні для JSON лінтера**

-   **`    --json-linter-enabled`**=_`<true|false>`_ &mdash;
    Контролювати лінтер для JSON (та його надмов) файлів.

**Глобальні опції, що застосовуються до всіх команд**

-   **`    --colors`**=_`<off|force>`_ &mdash;
    Встановити режим форматування для розмітки: "off" друкує все як звичайний текст, "force" примушує форматування розмітки за допомогою ANSI, навіть якщо консольний вихід визначено як несумісний
-   **`    --use-server`** &mdash;
    Підключитися до запущеного екземпляра серверу Biome daemon.
-   **`    --verbose`** &mdash;
    Друкувати додаткові діагностики, а деякі діагностики показують більше інформації. Також друкувати, які файли були оброблені та які з них були змінені.
-   **`    --config-path`**=_`PATH`_ &mdash;
    Встановити шлях до конфігураційного файлу або шлях до директорії для пошуку `biome.json` або `biome.jsonc`. Якщо використовується, вимикає стандартне вирішення конфігураційного файлу.
-   **`    --max-diagnostics`**=_`<none|<NUMBER>>`_ &mdash;
    Обмежити кількість відображуваних діагностик. Коли вказано `none`, ліміт знімається.
    [за замовчуванням: 20]
-   **`    --skip-errors`** &mdash;
    Пропускати файли з синтаксичними помилками замість видачі діагностики помилки.
-   **`    --no-errors-on-unmatched`** &mdash;
    Приглушити помилки, які будуть видані у випадку, якщо жоден файл не був оброблений під час виконання команди.
-   **`    --error-on-warnings`** &mdash;
    Вказати Biome виходити з кодом помилки, якщо деякі діагностики видають попередження.
-   **`    --reporter`**=_`<json|json-pretty|github|junit|summary|gitlab>`_ &mdash;
    Дозволяє змінити спосіб звітування діагностик та підсумків.
-   **`    --log-level`**=_`<none|debug|info|warn|error>`_ &mdash;
    Рівень логування. У порядку від найбільш детального до найменш детального: debug, info, warn, error.

    Значення `none` не показуватиме жодного логування.

    [за замовчуванням: none]

-   **`    --log-kind`**=_`<pretty|compact|json>`_ &mdash;
    Як повинен виглядати лог.
    [за замовчуванням: pretty]
-   **`    --diagnostic-level`**=_`<info|warn|error>`_ &mdash;
    Рівень діагностик для відображення. У порядку від найнижчого до найважливішого: info, warn, error. Передача `--diagnostic-level=error` призведе до того, що Biome друкуватиме лише діагностики, що містять лише помилки.
    [за замовчуванням: info]

**Доступні позиційні елементи:**

-   _`PATH`_ &mdash;
    Окремий файл, окремий шлях або список шляхів

**Доступні опції:**

-   **`    --write`** &mdash;
    Записує безпечні виправлення
-   **`    --unsafe`** &mdash;
    Дозволити небезпечні виправлення, слід використовувати з `--write` або `--fix`
-   **`    --fix`** &mdash;
    Псевдонім для `--write`, записує безпечні виправлення
-   **`    --apply`** &mdash;
    Псевдонім для `--write`, записує безпечні виправлення (застаріле, використовуйте `--write`)
-   **`    --apply-unsafe`** &mdash;
    Псевдонім для `--write --unsafe`, записує безпечні та небезпечні виправлення (застаріле, використовуйте `--write --unsafe`)
-   **`    --only`**=_`<GROUP|RULE>`_ &mdash;
    Запустити лише вказане правило або групу правил. Якщо рівень серйозності правила `off`, то рівень серйозності правила встановлюється на `error`, якщо це рекомендоване правило, або `warn` в іншому випадку.

    Приклад: `biome lint --only=correctness/noUnusedVariables --only=suspicious`

-   **`    --skip`**=_`<GROUP|RULE>`_ &mdash;
    Пропустити вказане правило або групу правил, встановивши рівень серйозності правил на `off`. Ця опція має пріоритет над `--only`.

    Приклад: `biome lint --skip=correctness/noUnusedVariables --skip=suspicious`

-   **`    --stdin-file-path`**=_`PATH`_ &mdash;
    Використовуйте цю опцію, коли хочете форматувати код, переданий через `stdin`, і вивести результат на `stdout`.

    Приклад: `echo 'let a;' | biome lint --stdin-file-path=file.js`

-   **`    --staged`** &mdash;
    Коли встановлено true, будуть перевірені лише файли, які були підготовлені до коміту.
-   **`    --changed`** &mdash;
    Коли встановлено true, будуть перевірені лише файли, які були змінені порівняно з вашою конфігурацією `defaultBranch`.
-   **`    --since`**=_`REF`_ &mdash;
    Використовуйте це для вказівки базової гілки для порівняння, коли ви використовуєте прапорець --changed і `defaultBranch` не встановлено у вашому biome.json
-   **`-h`**, **`--help`** &mdash;
    Друкує інформацію про допомогу

## biome format

Запустити форматувальник для набору файлів.

**Використання**: **`biome`** **`format`** \[**`--write`**\] \[**`--staged`**\] \[**`--changed`**\] \[**`--since`**=_`REF`_\] \[_`PATH`_\]...

**Загальні опції, що застосовуються до всіх файлів**

-   **`    --use-editorconfig`**=_`<true|false>`_ &mdash;
    Використовувати будь-які `.editorconfig` файли для налаштування форматувальника. Конфігурація в `biome.json` перевизначить конфігурацію `.editorconfig`. За замовчуванням: false.
-   **`    --indent-style`**=_`<tab|space>`_ &mdash;
    Стиль відступу.
-   **`    --indent-size`**=_`NUMBER`_ &mdash;
    Розмір відступу, за замовчуванням 2 (застаріле, використовуйте `indent-width`)
-   **`    --indent-width`**=_`NUMBER`_ &mdash;
    Розмір відступу, за замовчуванням 2
-   **`    --line-ending`**=_`<lf|crlf|cr>`_ &mdash;
    Тип закінчення рядка.
-   **`    --line-width`**=_`NUMBER`_ &mdash;
    Максимальна ширина рядка. За замовчуванням 80.
-   **`    --attribute-position`**=_`<multiline|auto>`_ &mdash;
    Стиль позиціонування атрибутів у HTML-подібних мовах. За замовчуванням auto.
-   **`    --bracket-spacing`**=_`<true|false>`_ &mdash;
    Чи вставляти пробіли навколо дужок в об'єктних літералах. За замовчуванням true.

**Опції форматування, специфічні для файлів JavaScript**

-   **`    --jsx-quote-style`**=_`<double|single>`_ &mdash;
    Тип лапок, що використовуються в JSX. За замовчуванням double.
-   **`    --quote-properties`**=_`<preserve|as-needed>`_ &mdash;
    Коли властивості в об'єктах цитуються. За замовчуванням asNeeded.
-   **`    --trailing-comma`**=_`<all|es5|none>`_ &mdash;
    Друкувати кінцеві коми де це можливо в багаторядкових синтаксичних структурах, розділених комами. За замовчуванням "all".
-   **`    --trailing-commas`**=_`<all|es5|none>`_ &mdash;
    Друкувати кінцеві коми де це можливо в багаторядкових синтаксичних структурах, розділених комами. За замовчуванням "all".
