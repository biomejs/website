# Розширення Biome для VS Code

[Biome](https://biomejs.dev/) об'єднує ваш стек розробки, поєднуючи функціональність окремих інструментів. Він використовує один файл конфігурації, має чудову продуктивність і працює з будь-яким стеком. Це розширення приносить Biome у ваш редактор, щоб ви могли:

- Форматувати файли *при збереженні* або при виконанні команди *Форматувати документ*
- Бачити помилки під час набору тексту та застосовувати виправлення коду
- Виконувати рефакторинг

## Встановлення

Ви можете встановити розширення, перейшовши на сторінку [Visual Studio Code Market Place](https://marketplace.visualstudio.com/items?itemName=biomejs.biome) або зсередини VS Code, виконавши одну з наступних дій:

- Відкрийте вкладку *розширення* (_Перегляд_ → _Розширення)_ і знайдіть Biome.
- Відкрийте _Швидке відкриття_ (<kbd>Ctrl</kbd>/<kbd title="Cmd">⌘</kbd>+<kbd>P</kbd> або _Перейти -> Перейти до файлу_), введіть `ext install biomejs.biome` і натисніть Enter.

## Початок роботи

### Форматувальник за замовчуванням

Налаштуйте Biome як форматувач за замовчуванням для підтримуваних файлів, щоб VS Code використовував Biome замість інших встановлених форматувачів. Ви можете зробити це, відкривши файл JavaScript або TypeScript, а потім:

- Відкрийте командну панель (<kbd>Ctrl</kbd>/<kbd title="Cmd">⌘</kbd>+<kbd title="Shift">⇧</kbd>+<kbd>P</kbd> або Перегляд → Командна панель)
- Виберіть _Форматувати документ за допомогою…_
- Виберіть _Налаштувати форматувач за замовчуванням…_
- Виберіть Biome

Ви також можете увімкнути Biome лише для певних мов:

- [Відкрийте `settings.json`](https://code.visualstudio.com/docs/getstarted/settings#_settingsjson): відкрийте _Командну панель_ (<kbd>Ctrl</kbd>/<kbd title="Cmd">⌘</kbd>+<kbd title="Shift">⇧</kbd>+<kbd>P</kbd>) і виберіть _Налаштування: Відкрити налаштування користувача (JSON)_
- І встановіть `editor.defaultFormatter` на `biomejs.biome` для бажаної мови

```json title="settings.json"
{
	"editor.defaultFormatter": "<інший форматувальник>",
	"[javascript]": {
		"editor.defaultFormatter": "biomejs.biome"
	}
}
```

Ця конфігурація встановлює Biome як форматувач за замовчуванням для файлів JavaScript. Усі інші файли будуть форматуватися за допомогою `<інший форматувальник>`.

## Розв'язання конфігурації

Розширення автоматично завантажує файл `biome.json` з кореневого каталогу робочої області.

## Розв'язання Biome

Розширення намагається використовувати Biome з локальних залежностей вашого проекту (`node_modules/@biomejs/biome`). Ми рекомендуємо додати Biome як залежність проекту, щоб скрипти NPM та розширення використовували одну й ту саму версію Biome.

Ви також можете явно вказати бінарний файл `Biome`, який має використовувати розширення, налаштувавши параметр `biome.lspBin` у налаштуваннях вашого редактора.

Якщо проект не має залежності від Biome і не вказано явний шлях, розширення використовує версію Biome, включену в його пакет.

## Використання

### Форматування документа

Щоб відформатувати весь документ, відкрийте _Командну панель_ (<kbd>Ctrl</kbd>/<kbd title="Cmd">⌘</kbd>+<kbd title="Shift">⇧</kbd>+<kbd>P</kbd>) і виберіть _Форматувати документ_.

Щоб відформатувати діапазон тексту, виділіть текст, який ви хочете відформатувати, відкрийте _Командну панель_ (<kbd>Ctrl</kbd>/<kbd title="Cmd">⌘</kbd>+<kbd title="Shift">⇧</kbd>+<kbd>P</kbd>), і виберіть _Форматувати виділення_.

### Форматування при збереженні

Biome враховує налаштування VS Code _Форматування при збереженні_. Щоб увімкнути форматування при збереженні, відкрийте налаштування (_Файл_ -> _Параметри_ -> _Налаштування_), знайдіть `editor.formatOnSave` і увімкніть цю опцію.

### Виправлення при збереженні

Biome враховує налаштування VS Code _Дії коду при збереженні_. Щоб увімкнути виправлення при збереженні, додайте


```json title="settings.json"
{
  "editor.codeActionsOnSave": {
    "quickfix.biome": "explicit"
  }
}
```

до файлу `settings.json` для VS Code.

### Сортування імпортів [Експериментально]

Розширення Biome для VS Code підтримує сортування імпортів через дію "Організувати імпорти". За замовчуванням цю дію можна виконати за допомогою комбінації клавіш <kbd title="Shift">⇧</kbd>+<kbd>Alt</kbd>+<kbd>O</kbd>, або вона доступна через _Командну панель_ (<kbd>Ctrl</kbd>/<kbd title="Cmd">⌘</kbd>+<kbd title="Shift">⇧</kbd>+<kbd>P</kbd>), вибравши _Організувати імпорти_.

Ви можете додати наступне до конфігурації вашого редактора, якщо хочете, щоб ця дія виконувалася автоматично при збереженні замість виклику вручну:

```json title="settings.json"
{
	"editor.codeActionsOnSave":{
		"source.organizeImports.biome": "explicit"
	}
}
```

## Налаштування розширення

### `biome.lspBin`

Опція `biome.lspBin` перевизначає бінарний файл Biome, який використовується розширенням.
Якщо шлях відносний, то базовим шляхом використовується робоча папка.

### `biome.rename`

Дозволяє Biome обробляти перейменування в робочій області (експериментально).

## Версіонування

Ми дотримуємося специфікацій, запропонованих [офіційною документацією](https://code.visualstudio.com/api/working-with-extensions/publishing-extension#prerelease-extensions):

Непарні мінорні версії призначені для попередніх випусків, наприклад, `*.5.*`.
Парні мінорні версії призначені для офіційних випусків, наприклад, `*.6.*`.

## Вирішення проблем

### Я встановив `@biomejs/biome`, але розширення показує попередження про те, що не вдалося знайти бібліотеку

Бібліотека `@biomejs/biome` вказує деякі необов'язкові залежності, які встановлюються залежно від вашої ОС та архітектури.

Можливо, що розширення не може знайти бінарний файл під час завантаження розширення. Це, ймовірно, викликано вашим менеджером пакетів.

**Щоб вирішити проблему**, спробуйте встановити бінарний файл вручну. Попередження має показати вам бінарний файл, який відповідає вашій машині.

**Якщо ви працюєте в команді, яка використовує різні ОС/архітектури**, рекомендується встановити всі бінарні файли:

- `@biomejs/cli-darwin-arm64`
- `@biomejs/cli-darwin-x64`
- `@biomejs/cli-linux-arm64`
- `@biomejs/cli-linux-x64`
- `@biomejs/cli-win32-arm64`
- `@biomejs/cli-win32-x64`

### Мій файл `biome.json` ігнорується в робочій області з кількома коренями

Наразі підтримка робочих областей з кількома коренями обмежена, що робить файли `biome.json`, розміщені в окремих кореневих папках, іноді невидимими для розширення. Поки що вам може знадобитися налаштувати окрему робочу область для кожної папки, яка залежить від Biome. Ви можете відстежувати наш прогрес у [цьому питанні](https://github.com/biomejs/biome/issues/1573).
