---
title: Інтеграція Biome з вашим VCS
description: Дізнайтеся, як інтегрувати Biome з VCS
---

Інтеграція з VCS (Система Контролю Версій) призначена для використання **додаткових** функцій, які може надати лише VCS.
На даний момент Biome підтримує лише Git.
Інтеграція є *опціональною**.
Ви повинні увімкнути `vcs.enabled` і встановити `vcs.clientKind` у конфігураційному файлі Biome:

```json title="biome.json"
{
  "vcs": {
    "enabled": true,
    "clientKind": "git"
  }
}
```

Ця конфігурація **сама по собі нічого не робить**. Вам потрібно увімкнути функції, які ви хочете використовувати.

### Використання файлу ігнорування

Увімкніть `vcs.useIgnoreFile`, щоб дозволити Biome ігнорувати всі файли та каталоги, зазначені у вашому файлі ігнорування VCS.
На даний момент Biome **враховує лише файл ігнорування у робочій директорії**.

```json title="biome.json" ins={5}
{
  "vcs": {
    "enabled": true,
    "clientKind": "git",
    "useIgnoreFile": true
  }
}
```

### Обробка лише змінених файлів

Ця функція доступна лише через CLI і дозволяє обробляти **лише** файли, які **змінилися** з однієї ревізії на іншу.

Спочатку вам потрібно оновити свій конфігураційний файл і вказати Biome, яка гілка є основною через поле `vcs.defaultBranch`:

```json title="biome.json" ins={6}
{
  "vcs": {
    "enabled": true,
    "clientKind": "git",
    "useIgnoreFile": true,
    "defaultBranch": "main"
  }
}
```

Потім додайте опцію `--changed` до своєї команди, щоб обробляти лише ті файли, які ваш VCS визнав "зміненими". Biome за допомогою VCS визначить змінені файли з гілки `main` та вашої поточної ревізії:

```shell
biome check --changed
```

:::warning
Biome не перевіряє, що змінилося, це означає, що навіть додавання пробілів або нових рядків до файлу позначить цей файл як "змінений"
:::

Альтернативно, ви можете використовувати опцію `--since`, щоб вказати довільну гілку. Ця опція **має пріоритет** над опцією `vcs.defaultBranch`. Наприклад, ви можете перевірити свої зміни відносно гілки `next`:

```shell
biome check --changed --since=next
```

### Обробка лише індексованих файлів

Перед тим, як зафіксувати свої зміни, ви можете перевірити форматування та лінти файлів, які були додані до _індексу_, також відомих як _індексовані файли_.
Додайте опцію `--staged` до своєї команди, щоб обробляти лише ці файли:

```shell
biome check --staged
```

Опція `--staged` недоступна для команди `ci`, оскільки ви не очікуєте фіксувати зміни в середовищі CI.
