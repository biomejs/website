---
title: Інтеграція Biome у ваш редактор
description: Дізнайтеся, як інтегрувати Biome з редакторами та IDE
---

Biome має першокласну підтримку [LSP](https://microsoft.github.io/language-server-protocol/). Якщо ваш редактор реалізує LSP, то інтеграція Biome повинна бути безпроблемною.

### Використовуйте LSP проксі

Biome має команду під назвою `lsp-proxy`. Коли вона виконується, Biome запускає два процеси:
- [демон](/internals/architecture#daemon), який виконує запитувані операції;
- сервер, який функціонує як проксі між запитами клієнта - редактора - та сервером - демоном;

Якщо ваш редактор може взаємодіяти з сервером і надсилати запити [JSON-RPC](https://www.jsonrpc.org/), вам потрібно лише налаштувати редактор для запуску цієї команди.

Ви можете перевірити, як це робить [плагін для neo-vim biome](https://github.com/neovim/nvim-lspconfig/blob/master/lua/lspconfig/configs/biome.lua).

### Використовуйте `stdin`

Якщо ваш редактор не підтримує LSP, ви можете використовувати бінарний файл `biome` безпосередньо і викликати його за допомогою [стандартного вводу](https://en.wikipedia.org/wiki/Standard_streams#Standard_input_(stdin)).

Наступні команди можна викликати через стандартний ввід:
- [`format`](/reference/cli/#biome-format)
- [`lint`](/reference/cli/#biome-lint)
- [`check`](/reference/cli/#biome-check)

Biome поверне новий вихідний результат (або оригінальний вихідний результат, якщо змін не відбулося) до [стандартного виводу](https://en.wikipedia.org/wiki/Standard_streams#Standard_output_(stdout)) і діагностику до [стандартної помилки](https://en.wikipedia.org/wiki/Standard_streams#Standard_error_(stderr)).

Коли ви використовуєте `stdin`, ви повинні передати опцію `--stdin-file-path`. Файл `path` **не обов'язково повинен існувати** у вашій файловій системі, це може бути будь-яке ім'я. **Важливо** надати правильне розширення файлу, щоб Biome знав, **як обробляти** ваш файл.

Відповідальність редактора полягає в тому, щоб знайти шлях до бінарного файлу та викликати його, коли це необхідно. Бінарні файли постачаються до npm на основі архітектур та ОС, які ми підтримуємо:

- `@biomejs/cli-darwin-arm64`
- `@biomejs/cli-darwin-x64`
- `@biomejs/cli-linux-arm64`
- `@biomejs/cli-linux-x64`
- `@biomejs/cli-win32-arm64`
- `@biomejs/cli-win32-x64`

Ім'я бінарного файлу - `biome` або `biome.exe`, і його можна знайти в кореневій директорії бібліотеки, наприклад: `@biomejs/cli-darwin-arm64/biome`, `@biomejs/cli-win32-x64/biome.exe`.

### Використовуйте демон з бінарним файлом

Використання бінарного файлу через CLI є дуже ефективним, хоча ви не зможете надати [журнали](#daemon-logs) вашим користувачам. CLI дозволяє вам запустити демон, а потім використовувати команди CLI через сам демон.

Для цього спочатку потрібно запустити процес демона за допомогою команди [`start`](/reference/cli#biome-start):

```shell
biome start
```
Потім кожна команда повинна додати опцію `--use-server`, наприклад:

```shell
echo "console.log('')" | biome format --use-server --stdin-file-path=dummy.js
```

:::note
Якщо ви вирішите використовувати демон, ви також несете відповідальність за перезапуск/зупинку процесу за допомогою команди [`stop`](/reference/cli#biome-stop), щоб уникнути появи привидів процесів.
:::

:::caution
Операції через демон значно повільніші, ніж сам CLI, тому рекомендується виконувати операції лише на окремих файлах.
:::

### Журнали демона

Демон Biome зберігає журнали у вашій файловій системі.
Вони зберігаються у папці під назвою `biome-logs`.
Шлях до цієї папки буде відрізнятися залежно від вашої операційної системи:

-  Linux: `~/.cache/biome`;
-  Windows: `C:\Users\<UserName>\AppData\Local\biomejs\biome\cache`
-  macOS: `/Users/<UserName>/Library/Caches/dev.biomejs.biome`

Для інших операційних систем ви можете знайти папку в тимчасовій директорії системи.

Щоб отримати точний шлях, виконайте наступну команду:

```shell
biome explain daemon-logs
```

Файли журналів обертаються щогодини.
Ви можете видалити журнали за допомогою команди `clean`:

```shell
biome clean
```
