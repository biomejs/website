---
title: Інтеграція Biome в розширення редактора
description: Дізнайтеся, як інтегрувати Biome з редакторами та IDE
---

Biome має першокласну підтримку [LSP](https://microsoft.github.io/language-server-protocol/). Якщо ваш редактор підтримує LSP, то інтеграція Biome повинна бути безшовною.

### Використання LSP проксі

Biome має команду під назвою `lsp-proxy`. При виконанні, Biome запускає два процеси:
- [демон](/internals/architecture#daemon), який виконує запитувані операції;
- сервер, який функціонує як проксі між запитами клієнта - редактора - та сервером - демоном;

Якщо ваш редактор може взаємодіяти з сервером і надсилати запити [JSON-RPC](https://www.jsonrpc.org/), вам потрібно лише налаштувати редактор для виконання цієї команди.

Ви можете перевірити, як це робить [`neo-vim biome extension`](https://github.com/neovim/nvim-lspconfig/blob/master/lua/lspconfig/configs/biome.lua).

### Використання `stdin`

Якщо ваш редактор не підтримує LSP, ви можете безпосередньо використовувати бінарний файл `biome` і викликати його за допомогою [стандартного вводу](https://en.wikipedia.org/wiki/Standard_streams#Standard_input_(stdin)).

Наступні команди можуть бути викликані через стандартний ввід:
- [`format`](/reference/cli/#biome-format)
- [`lint`](/reference/cli/#biome-lint)
- [`check`](/reference/cli/#biome-check)

Biome поверне новий вихідний результат (або оригінальний вихідний результат, якщо зміни не відбулися) до [стандартного виводу](https://en.wikipedia.org/wiki/Standard_streams#Standard_output_(stdout)) та діагностику до [стандартної помилки](https://en.wikipedia.org/wiki/Standard_streams#Standard_error_(stderr)).

Коли ви використовуєте `stdin`, ви повинні передати опцію `--stdin-file-path`. Файл `path` **не обов'язково має існувати** у вашій файловій системі, це може бути будь-яке ім'я. **Важливо** надати правильне розширення файлу, щоб Biome знав **як обробляти** ваш файл.

Відповідальність редактора - знайти шлях до бінарного файлу та викликати його, коли це необхідно. Бінарні файли постачаються до npm на основі архітектур та ОС, які ми підтримуємо:

- `@biomejs/cli-darwin-arm64`
- `@biomejs/cli-darwin-x64`
- `@biomejs/cli-linux-arm64`
- `@biomejs/cli-linux-x64`
- `@biomejs/cli-win32-arm64`
- `@biomejs/cli-win32-x64`

Назва бінарного файлу - `biome` або `biome.exe`, і його можна знайти в кореневому каталозі бібліотеки, наприклад: `@biomejs/cli-darwin-arm64/biome`, `@biomejs/cli-win32-x64/biome.exe`.

### Використання демона з бінарним файлом

Використання бінарного файлу через CLI дуже ефективне, хоча ви не зможете надати [журнали](#журнали-демона) своїм користувачам. CLI дозволяє вам запустити демон, а потім використовувати команди CLI через сам демон.

Щоб це зробити, спочатку потрібно запустити процес демона за допомогою команди [`start`](/reference/cli#biome-start):

```shell
biome start
```
Потім кожна команда повинна додати опцію `--use-server`, наприклад:

```shell
echo "console.log('')" | biome format --use-server --stdin-file-path=dummy.js
```

:::note
Якщо ви вирішите використовувати демона, ви також несете відповідальність за перезапуск/зупинку процесу за допомогою команди [`stop`](/reference/cli#biome-stop), щоб уникнути появи привидних процесів.
:::

:::caution
Операції через демона значно повільніші, ніж сам CLI, тому рекомендується виконувати операції лише на окремих файлах.
:::

### Журнали демона

Демон Biome зберігає журнали у вашій файловій системі. Журнали зберігаються в папці під назвою `biome-logs`. Шлях до цієї папки змінюється в залежності від вашої операційної системи:
-  Linux: `~/.cache/biome`;
-  Windows: `C:\Users\<UserName>\AppData\Local\biomejs\biome\cache`
-  macOS: `/Users/<UserName>/Library/Caches/dev.biomejs.biome`

Для інших операційних систем ви можете знайти папку в тимчасовому каталозі системи.

Щоб отримати точний шлях, виконайте наступну команду:
```shell
biome explain daemon-logs
```

Файли журналів обертаються щогодини.
