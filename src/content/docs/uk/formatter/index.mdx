Biome - це думковий форматувальник, який [підтримує кілька мов](/internals/language-support).
Він дотримується схожої [філософії до Prettier](https://prettier.io/docs/en/option-philosophy.html),
підтримуючи лише кілька опцій, щоб уникнути дебатів щодо стилів, які перетворюються на дебати щодо опцій Biome.
Він навмисно [опирається бажанню додавати нові опції](https://github.com/prettier/prettier/issues/40), щоб запобігти [обговоренням дрібниць](https://en.wikipedia.org/wiki/Law_of_triviality) в командах, щоб вони могли зосередитися на тому, що дійсно важливо.

## CLI

Наступна команда перевіряє форматування файлів у директорії `src`.
Вона виводить текстові відмінності, якщо знаходить код, який не відформатований.

<PackageManagerBiomeCommand command="format ./src" />

Якщо ви хочете **застосувати** нове форматування, додайте опцію `--write`:

<PackageManagerBiomeCommand command="format --write ./src" />

Команда приймає список файлів та директорій.

:::caution
Якщо ви передаєте глоб як параметр, ваша оболонка розширить його.
Результат розширення залежить від вашої оболонки.
Наприклад, деякі оболонки не підтримують рекурсивний глоб `**` або чергування `{}` у наступній команді:

```shell
biome format ./src/**/*.test.{js,ts}
```

Розширення оболонки має вартість продуктивності та обмеження на кількість файлів, які ви можете передати команді.
:::

Для отримання додаткової інформації про всі доступні опції, перегляньте [довідку CLI](/reference/cli#biome-format).

## Опції

Biome надає деякі опції для налаштування поведінки свого форматувача.
На відміну від інших інструментів, Biome розділяє опції, що не залежать від мови, від опцій, що залежать від мови.

Опції форматувача можна встановити через [CLI](/reference/cli/#biome-format) або через [конфігураційний файл Biome](/guides/configure-biome).
Починаючи з версії 1.9, Biome підтримує завантаження файлів `.editorconfig`.

Рекомендується використовувати [конфігураційний файл Biome](/guides/configure-biome), щоб забезпечити застосування однакових опцій як для Biome CLI, так і для Biome LSP.
Застосовуються такі значення за замовчуванням:

```json title="biome.json"
{
  "formatter": {
    "enabled": true,
    "formatWithErrors": false,
    "ignore": [],
    "attributePosition": "auto",
    "indentStyle": "tab",
    "indentWidth": 2,
    "lineWidth": 80,
    "lineEnding": "lf"
  },
  "javascript": {
    "formatter": {
      "arrowParentheses":"always",
      "bracketSameLine": false,
      "bracketSpacing": true,
      "jsxQuoteStyle": "double",
      "quoteProperties": "asNeeded",
      "semicolons": "always",
      "trailingCommas": "all"
    }
  },
  "json": {
    "formatter": {
      "trailingCommas": "none"
    }
  }
}
```

Основні опції, що не залежать від мови, які підтримує форматувач Biome:

- стиль відступів (за замовчуванням: `tab`): Використовувати пробіли або табуляцію для відступів;
- ширина відступів (за замовчуванням: `2`): Кількість пробілів на рівень відступу.
- ширина рядка (за замовчуванням: `80`): Ширина колонки, при якій Biome переносить код;

Дивіться [довідку з конфігурації](/reference/configuration#formatter) для отримання додаткових деталей.

## Ігнорування коду

Іноді відформатований код не є ідеальним.

Для таких випадків ви можете використовувати коментар для придушення форматування:

```js title="example.js"
// biome-ignore format: <пояснення>
```

Приклад:

```js title="example.js"
const expr =
  // biome-ignore format: масив не повинен бути відформатований
  [
    (2 * n) / (r - l),
    0,
    (r + l) / (r - l),
    0,
    0,
    (2 * n) / (t - b),
    (t + b) / (t - b),
    0,
    0,
    0,
    -(f + n) / (f - n),
    -(2 * f * n) / (f - n),
    0,
    0,
    -1,
    0,
  ];
```

Biome не надає коментарів для ігнорування всього файлу.
Однак, ви можете [ігнорувати файл за допомогою конфігураційного файлу Biome](/guides/configure-biome/#ignore-files).
