Biome 最推薦作為專案的開發相依套件 (development dependency) 進行安裝，但它也提供不需要 Node.js 的[獨立可執行檔]。

[獨立可執行檔]: /zh-tw/guides/manual-installation

<PackageManagerCommand
  npm="i -D -E @biomejs/biome"
  pnpm="add -D -E @biomejs/biome"
  yarn="add -D -E @biomejs/biome"
  bun="add -D -E @biomejs/biome"
  deno="add -D npm:@biomejs/biome"
/>

:::note[固定版本]
`-E` 參數可確保套件管理器將 Biome 的版本固定。請參閱
[版本控制頁面](/zh-tw/internals/versioning)
瞭解更多關於[為什麼固定版本很重要]的詳細資訊。
:::

[為什麼固定版本很重要]: /zh-tw/internals/versioning#why-pinning-the-version-is-important

## 設定

雖然 Biome 無需任何設定即可直接運行，但您可能希望調整某些設定以滿足專案需求。在此情況下，您可以執行以下指令生成 `biome.json` 設定檔。

<PackageManagerBiomeCommand command="init"/>

## 使用方法

讓我們快速瞭解如何在專案中使用 Biome。

### 編輯器整合

Biome 以官方擴充套件的形式支援您喜愛的編輯器。

- [VS Code](/zh-tw/editors/first-party-extensions#vs-code)
- [IntelliJ](/zh-tw/editors/first-party-extensions#intellij)
- [Zed](/zh-tw/editors/first-party-extensions#zed)

此外，還有許多適用於其他編輯器（例如 **Vim**、**Neovim** 及 **Sublime Text** 等）的[社群擴充套件](/zh-tw/editors/third-party-extensions)。

### 持續整合 (CI)

在 CI 流程中執行 `biome ci` 以在團隊間落實程式碼品質與一致性。它的運作方式與 `biome check` 指令類似，但專門針對 CI 環境進行了最佳化。

- [GitHub Actions](/zh-tw/recipes/continuous-integration#github-actions)
- [GitLab CI](/zh-tw/recipes/continuous-integration#gitlab-ci)

請參閱[持續整合](/zh-tw/recipes/continuous-integration)實用指南瞭解更多範例。
